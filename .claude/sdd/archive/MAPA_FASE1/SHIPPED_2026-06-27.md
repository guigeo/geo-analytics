## Summary

Aplicação web estática que renderiza cinco camadas geográficas oficiais do Brasil
(UF, município, bairro, setor censitário e antenas de telefonia) sobre um basemap
Protomaps auto-hospedado, com toggle de camadas e clique → atributos. Entrega o
valor da Fase 1 do PRD: a equipe de geomarketing passa a explorar recortes
territoriais e a base de antenas num mapa leve e reprodutível, sem GIS desktop nem
banco de dados. Estabelece também a **esteira de dados padronizada**
(`fonte heterogênea → GeoParquet → PMTiles`) que sustenta as Fases 2 (chat IA sobre
GeoParquet/DuckDB) e 3 (novos datasets sem refactor).

## Lessons Learned

### Process

- **Inspecionar os dados reais antes de escrever config evitou retrabalho.** Os campos
  no DESIGN eram suposições; `pyogrio.read_info` confirmou schemas, CRS e contagens
  (bairro 18k = moderada; setor 473k) antes do `datasets.yaml` — fechou o risco de
  AT-005/006 cedo.
- **Ship enxuto:** o polish visual (basemap detalhado, cores) foi conscientemente
  separado em feature futura para não sujar o milestone — a Fase 1 cumpre o escopo.

### Technical

- **Streaming via `ogr2ogr` foi decisivo:** converter o gpkg de 1.4 GB para GeoParquet
  levou ~30 s sem estouro de memória no Mac 16 GB. Confirma a Decisão 1 — GeoPandas
  in-memory teria falhado. O gargalo real é a tilagem (tippecanoe ~28 min), não a
  conversão.
- **Padrão `GeoParquet canônico + PMTiles uniforme`** funcionou: regra única sem
  ramificação por tamanho → adicionar dataset = entrada no YAML.
- **Bind mount vs venv:** colocar o venv em `/opt/venv` (`UV_PROJECT_ENVIRONMENT`)
  fora de `/repo` evitou que o mount do runtime sobrescrevesse as dependências.
- **GDAL `ubuntu-small` não tem driver Parquet** — precisa `ubuntu-full`.
- **Trade-off de basemap confirmado empiricamente:** zoom cheio do Brasil = 5.8 GB;
  `maxzoom: 8` → 20 MB. O detalhe de ruas custa tamanho (relevante para o VPS).

### Communication

- Confirmar incrementalmente (escopo → camadas → stack → basemap → Docker/VPS) deixou
  cada decisão rastreável; a padronização de formatos e o alvo VPS surgiram do diálogo
  e moldaram o design antes do build.

### Tools & Libraries

- `pyogrio` (GDAL embutido) para inspeção de schema sem instalar GDAL no host.
- `tippecanoe` + `pmtiles` (Protomaps) para tiles estáticos sem tile server.
- `uv` para ambiente Python isolado e rápido.
